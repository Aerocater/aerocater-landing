Security

AeroCater runs dispatch operations for real airport catering fleets. Here's how we protect the data behind it.

Architecture

  • Data encrypted at rest (AES-256) and in transit (TLS 1.3)
  • Row-level security enforced at the database layer — every query is scoped to the requesting organization, so one customer's data is never visible to another
  • Each airport station operates as a fully isolated organization, even when the same operator runs multiple stations

Authentication

  • Managed authentication via Supabase Auth, with session management and token expiry
  • Multi-factor authentication supported for admin and dispatcher accounts
  • Individual activity logging — every login and dispatch action is attributed to a specific user

Infrastructure

  • Hosted on Supabase (Postgres, Auth, Realtime, Storage), backed by AWS infrastructure in the United States
  • Automated daily backups
  • Static assets served via Netlify's global CDN with DDoS protection

Compliance

SOC 2 Type II certification is in progress. We'll publish our report here once available. In the meantime, we're happy to complete security questionnaires directly — contact security@aerocater.ca.

Vulnerability management

We run regular automated vulnerability scanning and plan an annual third-party penetration test covering the web app, API, and driver mobile flow. Remediation targets: Critical within 24 hours, High within 7 days, Medium within 30 days, Low within 90 days.

Responsible disclosure

Found a security issue? Email security@aerocater.ca with details. We aim to acknowledge reports within 48 hours.

Questions

Enterprise security review or DPA request: security@aerocater.ca
